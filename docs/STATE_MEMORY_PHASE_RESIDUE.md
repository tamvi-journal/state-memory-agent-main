# State Memory Positive Phase Residue (Minimal)

Positive phase residue is a lightweight state-memory trace for high-coherence runtime shifts.

## What it is

- A small advisory carryover hint attached to existing `StateMemoryRecord` flow.
- Captures bounded evidence fields only (`coherence_shift`, `trigger_cue`, `mode_shift`, `repair_event`).
- Uses event types such as `coherence_spike`, `positive_afterglow`, and `route_clarity_gain`.

## What it is not

- Not a second memory system.
- Not an authority override for gate, verification, wake_result, Tracey, monitor, or final synthesis.
- Not an emotional or subjective-state inference layer.

## Boundary

Positive residue records are observed/candidate memory artifacts only and remain advisory.
They are not auto-promoted to canonical by compression logic.

## Carryover and decay

Carryover decay strategy is intentionally deferred and should be introduced in a later, explicit update.
